# Trader Performance vs. Bitcoin Market Sentiment

Analysis of 211K Hyperliquid historical trades against the Bitcoin Fear & Greed Index — exploring whether
market sentiment predicts trader performance, and what that implies for trading strategy.

**Hiring assignment submission.** Full narrative write-up: [REPORT.pdf](REPORT.pdf) (also available as
[REPORT.md](REPORT.md)).
---

## Quick look

| | |
|---|---|
| Trades analyzed | 211,224 fills, 32 accounts, 246 coins |
| Sentiment data | 2,644 daily Fear & Greed Index readings (2018–2025) |
| Match rate | 100% of trades matched to a sentiment day |
| Headline finding | The day *after* an Extreme Fear reading produces the highest average next-day PnL of any condition studied ($72,701, ~2x the next-best), and it's statistically significant (p = 1.7×10⁻⁴) |

## Key findings

1. **Traders are contrarian, not momentum-driven** — 66% long-biased during Extreme Fear, flipping to 58%
   short-biased during Greed, with position size shrinking as greed rises.
2. **Extreme Fear and Extreme Greed are the most profitable regimes** by win rate (76–89%) and avg PnL/trade
   ($71–$130); Neutral is the weakest despite being "calm."
3. **Extreme Fear is a volume trap** — the most trades and volume of any regime, but the lowest win rate,
   meaning traders over-trade panic without a proportional payoff.
4. **Greed is the weakest regime risk-adjusted** (lowest Sharpe/Sortino), while **Extreme Greed is ~3x more
   capital-efficient** (218 bps PnL per dollar of volume) than any other regime.
5. **The tradeable signal is the day after Extreme Fear, not Extreme Fear itself** — see [REPORT.pdf](REPORT.pdf)
   §6 for the full predictive/lagged analysis.

All findings are backed by formal significance testing (Kruskal-Wallis, Mann-Whitney, chi-square) and
robustness checks (sample size per regime, outlier sensitivity, per-account PnL concentration) — see
[REPORT.pdf](REPORT.pdf) §4 and §7.

## Repository structure

```
.
├── README.md                      <- you are here
├── REPORT.md / REPORT.pdf         <- full write-up: methodology, findings, strategy, caveats
├── requirements.txt               <- Python dependencies
├── analysis.py                    <- reproducible analysis: stats, significance tests, risk
│                                     metrics, lagged/predictive analysis, robustness checks
├── build_pdf.py                   <- renders REPORT.pdf from the analysis results
├── data/
│   ├── historical_data.csv        <- Hyperliquid trade fills (source data)
│   └── fear_greed_index.csv       <- Bitcoin Fear & Greed Index (source data)
├── plot1–plot8_*.png              <- charts referenced in REPORT.pdf
└── *.csv                          <- intermediate result tables (stats, significance,
                                       risk-adjusted metrics, lagged analysis, robustness, etc.)
```

`merged_trades_sentiment.csv` (the trade-level table joined with daily sentiment, ~54 MB) is regenerated by
`analysis.py` and is git-ignored to keep the repo lean.

## How to reproduce

```bash
python3 -m pip install -r requirements.txt

python3 analysis.py         # runs the full analysis, writes all *.csv and plot*.png outputs
python3 build_pdf.py        # rebuilds REPORT.pdf from those outputs
python3 dashboard/build_data.py   # regenerates dashboard/data.js
```

All scripts use paths relative to their own location, so this works after cloning to any machine —
no path edits needed.

## Methodology notes

- Trades were joined to sentiment by calendar date (IST).
- Win-rate/avg-PnL statistics use only the ~104K fills with non-zero Closed PnL (the rest are
  position-opening fills without a realized PnL).
- Because trades on the same day aren't independent observations, every headline contrast is tested at
  both the trade level and the (statistically correct) day level — see REPORT.pdf §4 for where these
  disagree.
- No `leverage` field existed in this data export; leverage effects are proxied via position size
  (`Size USD`) only.

Full caveats and limitations are in [REPORT.pdf](REPORT.pdf) §9.
